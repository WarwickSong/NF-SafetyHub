# LLM-SafetyHub 云服务器迁移部署说明

> 该说明面向常见 Linux 云服务器，例如 Ubuntu、Debian、CentOS、Rocky Linux、AlmaLinux。  
> 如果云服务器是 Windows Server，可以优先参考现有 PowerShell 脚本；如果是 Linux，不要直接执行 `.ps1`，应使用本目录中的 `.sh` 脚本。

---

## 一、能否直接按当前目录操作

| 云服务器系统 | 是否可直接使用现有 PowerShell 脚本 | 推荐方式 |
|--------------|--------------------------------------|----------|
| Windows Server | 可以基本直接使用 `.ps1` | 使用 `setup_venv.ps1`、`deploy_direct.ps1` 或 `deploy_docker.ps1` |
| Linux | 不可以直接使用 `.ps1` | 使用 `setup_venv.sh`、`deploy_direct.sh` 或 `deploy_docker.sh` |
| Linux + Docker | 推荐 | 使用 Docker Compose 部署，运维最简单 |
| Linux + systemd | 可用 | 使用 `.venv` + `safetyhub.service.example` 托管服务 |

---

## 二、迁移前准备

### 2.1 云服务器基础要求

| 项目 | 建议 |
|------|------|
| 系统 | Ubuntu 22.04/24.04 LTS 或 Debian 12 |
| CPU | 当前阶段 1 核即可，后续中继流量增加建议 2 核以上 |
| 内存 | 当前阶段 1GB 可运行，后续建议 2GB 以上 |
| 磁盘 | 建议 20GB 以上，重点关注 `data/` 和日志增长 |
| 入站端口 | 当前测试可开放 `8080`；生产建议只开放 `80/443` |

### 2.2 上传项目

任选一种方式：

```bash
git clone <your-repo-url> /opt/NF-SafetyHub
```

或从本地打包上传：

```bash
scp -r NF-SafetyHub user@server:/opt/NF-SafetyHub
```

进入项目：

```bash
cd /opt/NF-SafetyHub
```

## 三、推荐方案：Linux Docker 部署

### 3.1 安装 Docker

Ubuntu/Debian 可参考：

```bash
sudo apt update
sudo apt install -y ca-certificates curl gnupg
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker $USER
```

重新登录 SSH 后确认：

```bash
docker --version
docker compose version
```

### 3.2 准备配置

```bash
cp 项目部署/.env.production.example .env
nano .env
```

至少修改：

```text
UPSTREAM_URL=真实上游地址
ADMIN_PASSWORD=至少12位强密码
```

### 3.3 启动

```bash
chmod +x 项目部署/*.sh
./项目部署/deploy_docker.sh
```

验证：

```bash
curl http://127.0.0.1:8080/health/live
curl http://127.0.0.1:8080/health/ready
```

## 四、可选方案：Linux 直接部署

### 4.1 安装 Python

Ubuntu/Debian：


### 4.2 初始化虚拟环境

### 4.3 准备配置

```bash

```

### 4.4 启动


验证：

```bash
curl http://127.0.0.1:8000/health/ready
```

## 五、systemd 托管直接部署

直接部署如果要后台常驻，建议用 systemd，不建议长期手动保持 SSH 会话。

### 5.1 复制服务文件

```bash
sudo cp 项目部署/safetyhub.service.example /etc/systemd/system/safetyhub.service
sudo nano /etc/systemd/system/safetyhub.service
```

确认路径、用户、工作目录正确。

### 5.2 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable safetyhub
sudo systemctl start safetyhub
sudo systemctl status safetyhub
```

查看日志：

```bash
journalctl -u safetyhub -f
```

## 六、云服务器安全检查

- `.env` 不能提交仓库，也不要发给无关人员。
- 生产环境必须使用强 `ADMIN_PASSWORD`。
- 当前 Compose 暴露 `8000` 和 `8080`，生产建议通过安全组/防火墙只开放必要端口。
- 当前阶段还没有完整认证后台和中继能力，公网部署只建议用于基础健康检查演示。
- SQLite 数据在 `data/`，迁移或备份时必须包含该目录。
- 生产 TLS、域名、证书续签、备份、监控属于阶段 3 加固范围。
