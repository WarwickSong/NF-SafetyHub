# LLM-SafetyHub 环境与部署总览

> 本目录集中存放环境初始化、本地运行、Windows 直接部署、Linux 云服务器部署、Docker 部署和 systemd 托管相关说明与脚本。

---

## 一、先看结论

如果迁移到云服务器：

| 云服务器类型 | 能否直接按当前目录操作 | 推荐入口 |
|--------------|------------------------|----------|
| Windows Server | 可以，使用 `.ps1` 脚本 | `直接部署说明.md`、`Docker部署说明.md` |
| Linux 云服务器 | 可以，使用 `.sh` 脚本 | `云服务器迁移部署说明.md` |
| Linux + Docker | 推荐 | `deploy_docker.sh` |
| Linux + systemd | 可用 | `safetyhub.service.example` |

当前项目处于阶段 1，检测引擎和中继引擎已完成；部署脚本可验证基础应用、健康检查、静态后台占位页、检测引擎测试和中继引擎测试。正式生产上线仍需要完成消息归档、审计、告警、管理后台和阶段 3 加固验收。

## 三、Windows 本地开发

```powershell
Copy-Item .\项目部署\.env.local.example .\.env
.\项目部署\setup_venv.ps1
.\项目部署\run_dev.ps1
```

访问：

```text
http://127.0.0.1:8000/health/live
http://127.0.0.1:8000/health/ready
```

验证：

```powershell
.\项目部署\verify_venv.ps1
```

## 六、Linux 云服务器 Docker 部署

推荐迁移到云服务器时优先使用 Docker Compose。

```bash
cd /opt/NF-SafetyHub
cp 项目部署/.env.production.example .env
nano .env
chmod +x 项目部署/*.sh
./项目部署/deploy_docker.sh
```

验证：

```bash
curl http://127.0.0.1:8080/health/live
curl http://127.0.0.1:8080/health/ready
```

## 八、端口说明

| 部署方式 | 默认端口 | 说明 |
|----------|----------|------|
| 直接部署 | `8000` | Uvicorn 直接监听 |
| Docker FastAPI | `8000` | Compose 暴露 FastAPI 容器端口 |
| Docker Nginx | `8080` | Compose 暴露 Nginx 端口 |
| 生产建议 | `80/443` | 阶段 3 再接入正式域名和 TLS |

## 九、注意事项

- `.env` 不应提交到仓库，根目录 `.gitignore` 已忽略该文件。
- `.venv` 不应提交到仓库，根目录 `.gitignore` 已忽略该目录。
- Linux 云服务器不要执行 `.ps1`，应执行 `.sh`。
- Windows 下首次尝试升级 pip 可能出现虚拟环境中断问题，当前推荐脚本不主动升级 pip，只安装项目依赖。
- 生产环境必须修改 `.env` 中的 `ADMIN_PASSWORD` 和 `UPSTREAM_URL`。
- 当前已实现 `/v1/chat/completions` 中继接口，但消息归档、审计、告警和完整管理后台尚未完成，公网部署仍只建议用于受控联调。
- 生产环境开放端口前，应先配置安全组、防火墙、认证、TLS 和日志/备份策略。
