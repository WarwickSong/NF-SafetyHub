# LLM-SafetyHub 直接部署说明

> 直接部署指不使用 Docker，直接通过项目根目录 `.venv` 运行 Uvicorn 服务。

---

## 一、适用场景

- 本地开发和调试。
- 内网测试机快速验证。
- 暂时不具备 Docker 环境的服务器。

---

## 二、准备环境

进入项目根目录：

```powershell
cd d:\Code\public\NF-SafetyHub
```

创建 `.env`：

```powershell
Copy-Item .\项目部署\.env.local.example .\.env
```

安装依赖：

```powershell
.\项目部署\setup_venv.ps1
```

---

## 三、启动开发服务

```powershell
.\项目部署\run_dev.ps1
```

默认监听：

```text
http://127.0.0.1:8000
```

健康检查：

```text
http://127.0.0.1:8000/health/live
http://127.0.0.1:8000/health/ready
```

---

## 四、直接部署启动

```powershell
Copy-Item .\项目部署\.env.production.example .\.env
notepad .\.env
.\项目部署\setup_venv.ps1
.\项目部署\deploy_direct.ps1
```

`deploy_direct.ps1` 会先初始化数据库，再用 `.venv` 中的 Uvicorn 启动服务。

---

## 五、验证命令

```powershell
.\.venv\Scripts\python.exe -m compileall .
.\.venv\Scripts\python.exe -m pytest
```

---

## 六、生产注意事项

- 当前 PowerShell 脚本适合 Windows 开发机和 Windows 测试机。
- Linux 生产环境建议使用 Docker Compose 或 systemd 单独托管 Uvicorn。
- 当前阶段还未实现完整中继接口，生产部署前必须完成阶段 1、阶段 2 和阶段 3 验收。
- 生产 `.env` 必须配置强密码，不能沿用示例值。
