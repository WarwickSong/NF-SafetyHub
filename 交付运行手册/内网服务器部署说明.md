# SafetyHub 内网服务器部署说明

## 适用场景

本说明适用于通过 VPN 和堡垒机访问的内网服务器部署，推荐使用 Docker Compose 同时部署 SafetyHub、PostgreSQL 和 Nginx。

除部署位置、数据盘路径和访问端口外，内网服务器配置应尽量与当前服务器一致，包括上游中转站 `https://yxai-api.nanfu.com`、Key Provider 类型、Provider 用户名、密码和认证版本。

交付目录只保留两个环境模板：

- `.env.current.example`：当前环境测试模板；
- `.env.production.example`：生产/内网部署模板。

默认对外端口为 8080，应用容器内部端口为 8000。需要对外使用 8000 时，只需要在 `.env` 中设置 `SAFETYHUB_HTTP_PORT=8000`。

## 一、源服务器导出 APIKeys

在当前源服务器执行：

```bash
cd /var/www/NF-SafetyHub
bash 交付运行手册/export_apikeys_for_intranet.sh
```

脚本会生成：

```text
data/export/safetyhub_apikeys_YYYYMMDD_HHMMSS.sql
data/export/safetyhub_apikeys_YYYYMMDD_HHMMSS.sql.sha256
```

该 SQL 只包含 `api_keys` 表数据，不包含审计、归档、图片、审批或后台操作记录。

## 二、打包代码

在 `/var/www` 下执行：

```bash
cd /var/www
tar --exclude='NF-SafetyHub/.git' \
    --exclude='NF-SafetyHub/.venv' \
    --exclude='NF-SafetyHub/.env' \
    --exclude='NF-SafetyHub/data/export' \
    --exclude='NF-SafetyHub/__pycache__' \
    --exclude='NF-SafetyHub/.pytest_cache' \
    -czf NF-SafetyHub-release.tar.gz NF-SafetyHub
```

把以下文件通过堡垒机传到内网服务器：

```text
NF-SafetyHub-release.tar.gz
safetyhub_apikeys_YYYYMMDD_HHMMSS.sql
```

## 三、内网服务器准备

在内网服务器执行：

```bash
cd /opt
tar -xzf NF-SafetyHub-release.tar.gz
cd /opt/NF-SafetyHub
cp 交付运行手册/.env.production.example .env
```

编辑 `.env`，把以下占位符改成当前服务器对应值或内网生产值：

```env
POSTGRES_PASSWORD=replace-with-strong-postgres-password
DB_URL=postgresql+asyncpg://safetyhub:replace-with-strong-postgres-password@postgres:5432/safetyhub
POSTGRES_DB_URL=postgresql+asyncpg://safetyhub:replace-with-strong-postgres-password@postgres:5432/safetyhub
SAFETYHUB_DATA_KEY=replace-with-strong-data-key
KEY_PROVIDER_PASSWORD=replace-with-current-key-provider-password
KEY_PROVIDER_AUTH_VERSION=replace-with-current-key-provider-auth-version
ADMIN_PASSWORD=replace-with-strong-admin-password
```

如果内网服务器的数据盘不是 `/data/safetyhub`，修改 `.env` 中这些配置：

```env
SAFETYHUB_DATA_ROOT=/data/safetyhub
SAFETYHUB_DATA_DIR=/data/safetyhub/app
SAFETYHUB_POSTGRES_DATA_DIR=/data/safetyhub/postgres
```

如果对外端口需要从 8080 改为 8000，修改：

```env
SAFETYHUB_HTTP_PORT=8000
```

首次部署前确认这些关键配置：

```env
ENVIRONMENT=production
DEBUG=false
UPSTREAM_URL=https://yxai-api.nanfu.com
KEY_PROVIDER_TYPE=oneapi_nanfu_yxai
KEY_PROVIDER_BASE_URL=https://yxai-api.nanfu.com
KEY_PROVIDER_USERNAME=800585
ALLOW_EMPTY_API_KEYS_PASSTHROUGH=false
```

## 四、自动部署并导入 APIKeys

```bash
cd /opt/NF-SafetyHub
bash 交付运行手册/deploy_intranet_docker.sh /opt/safetyhub_apikeys_YYYYMMDD_HHMMSS.sql
```

脚本会执行：

- 创建 `.env` 配置的数据盘目录；
- 启动 PostgreSQL、SafetyHub 和 Nginx；
- 清空目标库中的审计、归档、图片、审批、安全策略、后台操作等表；
- 在同一个 PostgreSQL 事务中清空目标 `api_keys` 并导入源端导出的 APIKeys；
- 输出 `docker compose ps` 和健康检查结果。

## 五、验证

```bash
docker compose ps
curl -i http://127.0.0.1:8080/health/ready
curl -I http://127.0.0.1:8080/admin/
```

如果 `.env` 中设置了 `SAFETYHUB_HTTP_PORT=8000`，验证端口也改成 8000。

浏览器通过 VPN 访问：

```text
http://内网服务器IP:8080/admin/
```

## 六、当前环境测试模板

如果只是复刻当前测试环境，可在目标机器执行：

```bash
cp 交付运行手册/.env.current.example .env
```

然后同样补齐 `KEY_PROVIDER_PASSWORD`、`KEY_PROVIDER_AUTH_VERSION` 等占位符。

## 七、常用运维命令

查看日志：

```bash
docker compose logs -f safetyhub
```

重启服务：

```bash
docker compose restart safetyhub nginx
```

停止服务：

```bash
docker compose down
```

只更新应用镜像并重启：

```bash
docker compose up --build -d safetyhub nginx
```

## 八、离线镜像场景

如果内网不能联网拉镜像，需要在可联网机器提前准备镜像：

```bash
docker compose build safetyhub
docker save postgres:16-alpine nginx:1.27-alpine nf-safetyhub-safetyhub -o safetyhub-images.tar
```

传到内网后执行：

```bash
docker load -i safetyhub-images.tar
```

再运行部署脚本。
