# LLM-SafetyHub 检测引擎运行说明

> 本文档说明阶段 1 第一批检测引擎的规则文件、验证方式和当前能力边界。

---

## 一、当前能力

检测引擎当前已具备：

- 文本归一化：URL 解码、HTML unescape、Unicode NFKC、零宽字符移除、控制字符清理、空白压缩。
- 关键词扫描：支持 `contains`、`exact`、`prefix` 匹配模式，支持大小写配置、启停规则、命中片段脱敏。
- 正则扫描：支持 YAML 规则加载、规则预编译、`ignore_case`、错误规则跳过、命中片段脱敏。
- 调度器：支持链式扫描、block 早停、warn 汇总、扫描器异常降级。
- 规则热加载入口：Scanner 提供 `reload()`，调度器提供 `reload_all()`。

---

## 二、规则文件

规则文件路径：

```text
engine/rules_config.yaml
```

当前规则数量：

| 类型 | 数量 | 说明 |
|------|------|------|
| 关键词规则 | 20 条 | 覆盖商业机密、凭据、内部信息、财务、人事、法务、安全等类型 |
| 正则规则 | 10 条 | 覆盖手机号、身份证、银行卡、邮箱、API Key、JWT、私钥、内网 IP 等类型 |

## 四、规则修改流程

1. 修改 `engine/rules_config.yaml`。
2. 执行规则配置测试：

```powershell
.\.venv\Scripts\python.exe -m pytest tests\test_rules_config.py
```

3. 执行检测引擎测试：

```powershell
.\.venv\Scripts\python.exe -m pytest tests\test_keyword.py tests\test_regex.py tests\test_scanner.py
```

4. 后续服务运行时，可通过 Scanner 的 `reload()` 或调度器的 `reload_all()` 重新加载规则。

---

## 五、当前边界

- 检测引擎已可独立运行测试，但尚未接入 `/v1/chat/completions`。
- 正则匹配当前使用 Python 标准库 `re`，未实现单条规则 100ms 强制超时。
- 当前规则命中片段已脱敏，但后续日志、审计、告警仍必须避免输出 prompt 原文和 APIKey 明文。
- 规则热加载已有代码入口，但尚未接入定时任务或管理后台按钮。

---

## 六、下一步接入点

阶段 1 第二批中继引擎实现后，需要在 FastAPI 生命周期中初始化：

```python
scanner = ScannerOrchestrator()
scanner.register(KeywordScanner(settings.rules_config_path))
scanner.register(RegexScanner(settings.rules_config_path))
app.state.scanner = scanner
```

随后在 `/v1/chat/completions` 中提取 messages 文本并调用：

```python
scan_result = await request.app.state.scanner.scan(extracted_text)
```
