# LLM-SafetyHub 规则配置说明

> 本文档说明 `engine/rules_config.yaml` 的配置格式和维护要求。

---

## 一、配置结构

```yaml
version: "0.2.0"
keyword_rules: []
regex_rules: []
```

---

## 二、关键词规则

示例：

```yaml
keyword_rules:
  - id: "KW-001"
    name: "商业机密-产品路线图"
    keywords: ["产品路线图", "roadmap", "产品规划"]
    level: "block"
    match_mode: "contains"
    case_sensitive: false
    enabled: true
    description: "拦截包含产品路线图相关关键词的对话"
```

字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 规则唯一 ID，建议 `KW-数字` |
| `name` | 是 | 规则名称 |
| `keywords` | 是 | 关键词列表 |
| `level` | 是 | `block`、`warn` 或 `pass` |
| `match_mode` | 否 | `contains`、`exact`、`prefix`，默认 `contains` |
| `case_sensitive` | 否 | 是否大小写敏感，默认 `false` |
| `enabled` | 否 | 是否启用，默认 `true` |
| `description` | 否 | 规则说明 |

---

## 三、正则规则

示例：

```yaml
regex_rules:
  - id: "RG-001"
    name: "PII-中国手机号"
    pattern: "(?<!\\d)1[3-9]\\d{9}(?!\\d)"
    level: "block"
    enabled: true
    description: "检测中国大陆手机号"
```

字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 规则唯一 ID，建议 `RG-数字` |
| `name` | 是 | 规则名称 |
| `pattern` | 是 | Python `re` 兼容正则表达式 |
| `level` | 是 | `block`、`warn` 或 `pass` |
| `enabled` | 否 | 是否启用，默认 `true` |
| `ignore_case` | 否 | 是否忽略大小写，默认 `false` |
| `description` | 否 | 规则说明 |

---

## 四、规则级别

| 级别 | 行为 | 当前状态 |
|------|------|----------|
| `block` | 拦截请求，后续返回伪装回复 | 检测引擎已输出 blocked，拦截动作待中继接入 |
| `warn` | 放行请求，记录审计和告警 | 检测引擎已输出 warned，审计告警待后续接入 |
| `pass` | 命中但不拦截 | 预留 |

---

## 五、维护要求

- 新增规则后必须运行 `tests/test_rules_config.py`。
- 规则 ID 必须唯一。
- 正则表达式应避免灾难性回溯。
- 命中片段会脱敏，但规则名称和描述仍不应包含真实密钥、客户隐私或生产凭据。
- 生产规则调整应先以 `warn` 观察，再逐步切换到 `block`。
