# LLM-SafetyHub 中继引擎运行说明

> 本文档说明阶段 1 第二批中继引擎的当前能力、配置方式、验证方式和边界。

---

## 一、当前能力

中继引擎当前已具备：

- `/v1/chat/completions` 路由入口。
- 请求 `messages` 文本提取，并接入检测引擎扫描。
- 命中 block 规则时返回 OpenAI Chat Completions 兼容伪装回复。
- 未命中 block 时转发请求到默认上游。
- 非流式响应 JSON 透传。
- 流式响应 SSE 逐 chunk 透传工具。
- Header 透传/剥离策略。
- 单上游路由，并预留模型、APIKey、capability 路由参数。

---

## 二、关键配置

生产或联调前，需要在 `.env` 中配置：

```text
UPSTREAM_URL=https://yxai-api.nanfu.com
UPSTREAM_TIMEOUT_CONNECT=10
UPSTREAM_TIMEOUT_READ=120
```

注意：

- `UPSTREAM_URL` 必须是上游中转站基础地址。
- 真实上游联调前，应确认 Authorization Header 会由客户端传入并按策略透传。
- 当前默认使用单上游，多上游策略后续启用。

## 四、手动联调示例

启动服务：

```powershell
.\交付运行手册\run_dev.ps1
```

发送正常请求并触达上游：

```powershell
$secureToken = Read-Host -AsSecureString "Input upstream token"
$tokenPtr = [Runtime.InteropServices.Marshal]::SecureStringToBSTR($secureToken)
try {
    $token = [Runtime.InteropServices.Marshal]::PtrToStringBSTR($tokenPtr)
    $body = '{"model":"gpt-test","messages":[{"role":"user","content":"hello"}]}'
    Invoke-RestMethod -Uri http://127.0.0.1:8000/v1/chat/completions -Method Post -ContentType "application/json; charset=utf-8" -Headers @{Authorization="Bearer $token"} -Body $body
} finally {
    [Runtime.InteropServices.Marshal]::ZeroFreeBSTR($tokenPtr)
}
```

发送命中 block 的请求：

```powershell
$body = '{"model":"gpt-test","messages":[{"role":"user","content":"\u8bf7\u5e2e\u6211\u5916\u53d1\u4ea7\u54c1\u8def\u7ebf\u56fe"}]}'
Invoke-RestMethod -Uri http://127.0.0.1:8000/v1/chat/completions -Method Post -ContentType "application/json; charset=utf-8" -Headers @{Authorization="Bearer placeholder-token"} -Body $body
```

命中 block 时不会请求上游，会直接返回伪装回复。PowerShell 命令行直接传中文可能发生编码损坏，建议使用 UTF-8 请求体文件或 JSON Unicode 转义。

## 五、Header 策略

默认剥离：

- Hop-by-hop Headers，例如 `Connection`、`Transfer-Encoding`、`Upgrade`。
- `Host`、`Cookie`、`Content-Length`。
- `X-SafetyHub-*`、`X-Admin-*`、`X-Internal-*` 等内部 Header。
- `X-Forwarded-*`、`X-Real-IP` 等由代理层重新设置的 Header。

默认透传：

- `Authorization`
- `Content-Type`
- 其他非高风险业务 Header
- `X-Request-ID` 会生成或透传后发送到上游

## 六、当前边界

- 当前尚未写入消息归档、审计和告警。
- 流式响应已具备透传工具，但完整流式归档将在下一批实现。
- 当前测试使用 `httpx.MockTransport` 模拟上游，真实联调需要配置有效 `UPSTREAM_URL`。
- 当前 block 伪装回复不会泄露命中规则名称或敏感片段。
