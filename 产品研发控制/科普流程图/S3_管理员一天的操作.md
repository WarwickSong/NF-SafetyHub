# S3. 管理员一天的操作

> 管理员上班后会做什么——按从轻到重的顺序串起来。

```mermaid
flowchart LR
    Login(["登录后台<br/>用户名 + 密码"])
    Login --> Dash["① 看仪表盘<br/>今天总请求 / 命中 / 拦截"]
    Dash --> Arch["② 看训练样本<br/>哪些通过请求沉淀成样本"]
    Arch --> Audit["③ 看审计记录<br/>哪些请求命中了什么规则"]
    Audit --> Rule["④ 调整规则<br/>开关关键词 / 增减正则<br/>一键热加载"]
    Rule --> Keys["⑤ 管 APIKey<br/>新增 / 替换 / 吊销 / CSV 批量"]
    Keys --> Runtime["⑥ 看运行状态<br/>排队多不多 / 队列丢没丢"]

    classDef step fill:#e3f2fd,stroke:#1976d2,color:#000
    class Dash,Arch,Audit,Rule,Keys,Runtime step
```

## 每一步在解决什么

| 步骤 | 解决的问题 |
|------|-----------|
| ① 仪表盘 | 今天系统忙不忙、有没有异常 |
| ② 训练样本 | 查看通过请求是否形成可用的 messages + assistant response；被拦截和脱敏证据看审计 |
| ③ 审计 | 看到底是哪条规则在拦人，是不是误伤 |
| ④ 规则 | 误伤了就放宽，漏了就收紧，**改完立即生效，不用重启** |
| ⑤ APIKey | 员工入职/离职、Key 泄露时快速处理 |
| ⑥ 运行状态 | 压力大不大、要不要扩容 |
