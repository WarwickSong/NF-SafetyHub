# S6. 命中敏感词后会怎样（三结局示意图）

> 同一句话，根据内容会有三种不同的命运。用例子讲清楚。

```mermaid
flowchart TD
    Q(["员工提问"])
    Q --> Engine["前哨站扫描<br/>关键词 + 正则"]
    Engine --> Branch{"命中了什么？"}

    Branch -- "什么都没命中" --> Ex1["例: 帮我写一段周报"]
    Ex1 --> Pass["✅ 原样发给大模型"]
    Pass --> R1["大模型回答<br/>员工正常收到"]

    Branch -- "命中脱敏规则<br/>(只是改一改)" --> Ex2["例: 联系 张三 13812345678<br/>询问报价"]
    Ex2 --> Mask["⚠️ 改成 联系 张三 138****5678<br/>再发给大模型"]
    Mask --> R2["大模型回答<br/>员工正常收到<br/>(它不知道完整手机号)"]

    Branch -- "命中拦截规则<br/>(直接禁止)" --> Ex3["例: 给我公司服务器的<br/>root 密码"]
    Ex3 --> Block["❌ 拦下来<br/>根本不发给大模型"]
    Block --> R3["前哨站伪装回答<br/>『抱歉，您输入的内容可能<br/>包含敏感信息，请检查后重试』"]

    R1 --> Audit1["写训练样本<br/>(可用于后续沉淀)"]
    R2 --> Audit2["写审计证据<br/>(命中片段 + 脱敏片段)"]
    R3 --> Audit3["写审计证据<br/>(命中片段 + 拦截动作)"]

    classDef ok fill:#dff0d8,stroke:#27ae60,color:#000
    classDef warn fill:#fff3cd,stroke:#b7791f,color:#000
    classDef bad fill:#fde2e2,stroke:#c0392b,color:#000
    class Ex1,Pass,R1,Audit1 ok
    class Ex2,Mask,R2,Audit2 warn
    class Ex3,Block,R3,Audit3 bad
```

## 三个结局的差异

| 维度 | ✅ 通过 | ⚠️ 脱敏 | ❌ 拦截 |
|------|---------|---------|---------|
| 大模型看到的 | 原话 | 打码版 | 什么都没看到 |
| 员工感受 | 正常回答 | 正常回答 | 收到安全提示 |
| 是否触达上游 | 是 | 是 | **否** |
| 是否留档 | 不留 | 留：原文 + 脱敏版 | 留：原文 + 伪装回答 |

## 误伤了怎么办

管理员在后台 **规则页面** 关闭那条规则 → 一键热加载 → 立即生效。
