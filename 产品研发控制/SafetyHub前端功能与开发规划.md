# LLM-SafetyHub 前端功能与开发规划

> 本文档定义 SafetyHub 管理员前端的页面范围、功能边界、接口依赖、开发阶段和验收标准。前端定位为轻量管理后台，服务于安全管理员查看数据、处理告警、管理规则和后续扩展 APIKey/模型权限、审批、文件安全能力。

---

## 一、前端定位

SafetyHub 前端是 **管理员管理后台**，不面向普通终端用户。核心目标是：

1. **可视化安全态势** — 展示请求量、拦截数、告警数、活跃用户、规则命中趋势
2. **查询与追溯** — 查询消息归档、审计日志、拦截记录，支持详情查看和导出
3. **规则运营** — 查看规则、启停规则、观察命中效果，为后续规则编辑预留入口
4. **告警与审批** — 查看告警记录、审批记录，为临时放行审批预留操作入口
5. **权限治理预留** — 为 APIKey、模型权限、多上游路由预留页面和接口字段
6. **系统配置** — 展示运行状态、Webhook、数据保留、健康检查、版本信息等配置

## 二、技术方案

### 2.1 技术选型

| 项目 | 方案 | 说明 |
|------|------|------|
| 前端形态 | 静态 HTML + 原生 JavaScript + CSS | MVP 阶段不引入 React/Vue，降低构建和部署复杂度 |
| 托管方式 | Nginx 静态文件 | `admin/static/` 挂载到 Nginx，API 由 FastAPI 提供 |
| API 前缀 | `/admin/api/*` | 所有管理接口统一前缀 |
| 认证方式 | Basic Auth + IP 白名单 | v1.0 使用轻量认证，后续可替换 SSO |
| 样式方案 | 单 CSS 文件 | `admin/static/css/style.css` |
| 公共逻辑 | 单 JS 文件 | `admin/static/js/app.js` |

### 2.2 目录结构

```
admin/static/
├── index.html          # 仪表盘
├── blocks.html         # 拦截记录
├── archives.html       # 消息归档
├── audits.html         # 审计日志
├── rules.html          # 规则管理
├── approvals.html      # 临时审批记录预留
├── api_keys.html       # APIKey/模型权限预留
├── files.html          # 文件安全预留
├── settings.html       # 系统设置
├── css/
│   └── style.css
└── js/
    └── app.js
```

## 四、页面功能定义

### 4.1 仪表盘

**目标**：让管理员快速了解系统运行状态和安全态势。

| 模块 | 内容 |
|------|------|
| 核心指标卡 | 今日请求数、今日拦截数、今日告警数、活跃用户数、归档失败数 |
| 趋势图 | 最近 7 天请求趋势、拦截趋势、告警趋势 |
| Top 列表 | Top 命中规则、Top 风险用户、Top 模型 |
| 系统状态 | 服务状态、数据库状态、规则加载状态、Webhook 状态 |
| 快捷入口 | 查看拦截记录、查看消息归档、查看规则、系统设置 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/stats` | 获取核心统计和趋势 |
| `GET /admin/api/health` | 获取管理端展示用健康状态 |
| `GET /admin/api/rules/top` | 获取 Top 命中规则，可后续合并到 stats |

**验收标准**

- [ ] 页面加载后展示今日请求数、拦截数、告警数、活跃用户
- [ ] 趋势数据为空时展示空状态，不报错
- [ ] 后端异常时展示错误提示，不泄露堆栈信息

---

### 4.2 拦截记录

**目标**：查看所有 block/warn 安全事件，支持定位用户、规则和请求。

| 功能 | 说明 |
|------|------|
| 列表展示 | 时间、用户、APIKey 前后缀、模型、规则、级别、动作、请求 ID |
| 筛选 | 用户、规则、级别、模型、时间范围 |
| 详情 | 脱敏命中片段、请求哈希、关联归档、告警状态 |
| 跳转 | 跳转到消息归档详情、审计详情 |
| 导出 | v1.0 可后置，v1.1 支持 CSV 导出 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/audits` | 查询 block/warn 事件 |
| `GET /admin/api/audits/{id}` | 查看事件详情 |
| `GET /admin/api/audits/export` | 导出审计记录，v1.1 启用 |

**验收标准**

- [ ] 支持分页，默认每页 20 条，最大 100 条
- [ ] 命中片段默认脱敏展示
- [ ] 查询详情和导出行为写入管理员操作审计

---

### 4.3 消息归档

**目标**：查询完整对话记录，支持安全追溯和问题排查。

| 功能 | 说明 |
|------|------|
| 列表展示 | 时间、用户、APIKey 前后缀、模型、stream、是否拦截、耗时 |
| 筛选 | 用户、模型、是否拦截、关键词、时间范围 |
| 详情查看 | prompt、response、token、延迟、关联审计事件 |
| 脱敏策略 | 列表默认不展示 prompt/response 原文，详情页按权限展示 |
| 操作审计 | 查看完整 prompt/response 必须记录管理员操作日志 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/archives` | 分页查询消息归档 |
| `GET /admin/api/archives/{id}` | 查看归档详情 |
| `GET /admin/api/archives/stats` | 查询归档统计 |

**验收标准**

- [ ] 列表页不展示完整 prompt/response
- [ ] 详情页支持复制 request_id，不支持直接复制完整 APIKey
- [ ] 查看详情动作写入管理员操作审计

---

### 4.4 审计日志

**目标**：统一查看安全事件审计和管理员操作审计。

| 类型 | 内容 |
|------|------|
| 安全审计 | 规则命中、拦截、告警、审批放行、文件扫描 |
| 管理员操作审计 | 登录、查看详情、导出、规则变更、审批操作、系统设置查看 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/audits` | 安全审计列表 |
| `GET /admin/api/admin-ops` | 管理员操作审计列表 |
| `GET /admin/api/audits/export` | 安全审计导出 |

**验收标准**

- [ ] 安全审计和管理员操作审计可切换查看
- [ ] 审计记录不可在前端删除或修改
- [ ] 导出操作写入管理员操作审计

---

### 4.5 规则管理

**目标**：展示规则库状态和命中效果，支持后续规则运营。

| 功能 | v1.0 | v1.1+ |
|------|------|-------|
| 规则列表 | 查看规则 ID、名称、类型、级别、状态 | 支持新增、编辑、禁用 |
| 规则详情 | 查看关键词/正则、描述、命中次数 | 支持测试样例和误报标记 |
| 热加载状态 | 展示规则文件版本和加载时间 | 支持前端触发 reload |
| 版本管理 | 展示当前规则版本 | 支持回滚和变更记录 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/rules` | 查询规则列表 |
| `GET /admin/api/rules/{id}` | 查询规则详情 |
| `POST /admin/api/rules/reload` | 触发规则热加载，v1.1 启用 |

**验收标准**

- [ ] v1.0 至少支持查看规则列表和状态
- [ ] 规则内容涉及敏感样例时默认脱敏展示
- [ ] 修改、启停、热加载动作必须写入管理员操作审计

### 4.6 系统设置

**目标**：集中展示系统配置和运行状态，v1.0 以只读为主。

| 模块 | 内容 |
|------|------|
| 基础信息 | 版本、启动时间、运行环境、Git commit |
| 上游配置 | 默认中转站、超时、当前路由模式 |
| 告警配置 | Webhook 类型、是否配置、频率限制参数 |
| 数据治理 | 归档保留天数、审计保留天数、清理任务状态 |
| 安全配置 | 管理后台认证、IP 白名单、文件上传开关、请求大小限制 |
| 健康状态 | live/ready、数据库、规则、Webhook |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/settings` | 获取脱敏后的系统配置 |
| `GET /admin/api/health` | 获取健康状态 |
| `POST /admin/api/webhook/test` | 测试 Webhook，v1.1 启用 |

**验收标准**

- [ ] 不展示 Webhook URL、admin password、上游 Key 等敏感原文
- [ ] 未配置项明确展示“未配置”或“已关闭”
- [ ] 设置页不允许 v1.0 直接修改生产配置

### 4.7 APIKey/模型权限

**目标**：为 APIKey、模型权限、多上游路由预留管理入口。

| 功能 | v1.0 | v1.1+ |
|------|------|-------|
| APIKey 列表 | 只读占位，展示字段定义 | 新增、禁用、过期、额度 |
| 模型权限 | 展示 allowlist 设计说明 | 按用户/APIKey/部门配置模型权限 |
| 能力权限 | 展示 capability 维度 | 控制 chat、file_upload、function_call、mcp_tool、reasoning |
| 上游映射 | 展示默认单上游 | 配置 APIKey/模型到不同中转站 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/api-keys` | APIKey 列表，v1.1 启用 |
| `GET /admin/api/model-policies` | 模型权限策略，v1.1 启用 |
| `GET /admin/api/upstream-routes` | 上游路由策略，v1.1 启用 |

**验收标准**

- [ ] v1.0 页面存在，但明确标记“预留能力，默认关闭”
- [ ] APIKey 只展示前后缀，不展示明文
- [ ] 后续启用时所有变更写入管理员操作审计

### 4.8 审批记录

**目标**：为敏感请求临时放行审批提供查看和处理入口。

| 功能 | v1.0 | v1.1+ |
|------|------|-------|
| 审批列表 | 只读占位 | 展示 pending/approved/rejected/expired/used |
| 审批详情 | 展示流程说明 | 展示脱敏片段、规则、用户、模型、请求哈希 |
| 审批操作 | 不启用 | 支持 approve/reject，生成一次性令牌 |
| 通知联动 | 不启用 | 飞书/企微交互式卡片联动 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/approvals` | 查询审批记录 |
| `GET /admin/api/approvals/{id}` | 查询审批详情 |
| `POST /admin/api/approvals/{id}/approve` | 审批通过，v1.1 启用 |
| `POST /admin/api/approvals/{id}/reject` | 审批拒绝，v1.1 启用 |

**验收标准**

- [ ] v1.0 页面只做占位说明，不影响核心链路
- [ ] 审批消息和页面都不展示完整敏感原文
- [ ] 审批通过令牌必须一次性、短有效期、绑定请求哈希

### 4.9 文件安全

**目标**：为上传文件解析、扫描、拦截、脱敏提供管理入口。

| 功能 | v1.0 | v1.1+ |
|------|------|-------|
| 文件扫描记录 | 只读占位 | 展示文件名、类型、大小、哈希、扫描结果 |
| 文件详情 | 展示能力说明 | 展示抽取文本摘要、命中规则、处理动作 |
| 脱敏文件 | 不启用 | 展示脱敏状态和脱敏文件哈希 |
| 不可解析文件 | 展示策略 | 支持拒绝、审批或白名单放行 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/files` | 文件扫描记录列表 |
| `GET /admin/api/files/{id}` | 文件扫描详情 |

**验收标准**

- [ ] v1.0 页面只做占位说明，文件上传能力默认关闭或白名单启用
- [ ] 文件原文不在列表页展示
- [ ] 文件哈希、扫描动作、处理结果可审计

## 五、公共交互规范

### 5.1 列表页规范

| 项目 | 要求 |
|------|------|
| 分页 | 默认 20 条/页，最大 100 条/页 |
| 筛选 | 筛选条件同步到 URL query，刷新不丢失 |
| 空状态 | 无数据时展示明确提示 |
| 错误状态 | 接口异常时展示错误，不显示后端堆栈 |
| 加载状态 | 请求中展示 loading，不重复触发查询 |
| 时间展示 | 后端存 UTC，前端按本地时区展示 |

### 5.2 敏感信息展示规范

| 信息 | 展示规则 |
|------|----------|
| APIKey | 只展示前 6 位和后 4 位，中间脱敏 |
| Webhook URL | 只展示是否已配置，不展示 URL 原文 |
| admin password | 永不展示 |
| prompt/response | 列表不展示原文，详情页按权限展示并记录操作审计 |
| matched_snippet | 始终脱敏展示 |
| 文件内容 | 默认不展示原文，只展示抽取摘要和哈希 |

### 5.3 操作审计规范

以下前端行为必须触发管理员操作审计：

- 查看完整消息归档详情
- 导出审计日志或归档数据
- 修改规则、启停规则、触发规则热加载
- 测试 Webhook
- 审批通过或拒绝
- 修改 APIKey、模型权限、上游路由
- 查看文件扫描详情或下载脱敏文件

## 六、接口契约

### 6.1 通用响应格式

```json
{
  "data": {},
  "request_id": "req_xxx",
  "message": "ok"
}
```

### 6.2 分页响应格式

```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "page_size": 20,
  "request_id": "req_xxx"
}
```

### 6.3 错误响应格式

```json
{
  "error": {
    "code": "FORBIDDEN",
    "message": "无权限访问该资源"
  },
  "request_id": "req_xxx"
}
```

### 6.4 前端必须处理的错误码

| 错误码 | 前端行为 |
|--------|----------|
| `UNAUTHORIZED` | 提示重新登录 |
| `FORBIDDEN` | 展示无权限提示 |
| `NOT_FOUND` | 展示资源不存在 |
| `VALIDATION_ERROR` | 展示参数错误 |
| `RATE_LIMITED` | 提示稍后重试 |
| `INTERNAL_ERROR` | 展示通用错误，附 request_id |
