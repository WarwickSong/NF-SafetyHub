# LLM-SafetyHub 前端功能与开发规划

> 本文档定义 SafetyHub 管理员前端的页面范围、功能边界、接口依赖、开发阶段和验收标准。前端定位为轻量管理后台，当前开发范围收敛到阶段 6A 单实例 Docker 生产稳定性与高并发治理；阶段 7 及之后前端增强暂不开发，仅作为长期规划保留。模型权限、token 额度和资源能力权限由中转站管理，不作为 SafetyHub 前端默认权限配置对象。

---

## 一、前端定位

SafetyHub 前端是 **管理员管理后台**，不面向普通终端用户。核心目标是：

1. **可视化安全态势** — 展示阶段 6 及之前已有统计数据，告警数和趋势增强仅保留为长期规划；阶段 6A 后台统计需支持短缓存，避免压测期间实时重查询拖慢后台
2. **生产运行观测** — 阶段 6A 增加单实例运行状态展示口径，包括 `/v1/*` 在途数、排队数、队列拒绝数、归档队列状态和上游连接池摘要；默认 4 worker 下展示面向 100 名员工峰值场景的容器总 `1000` in-flight + `2000` 排队目标及 `.env` 实际配置
3. **查询与追溯** — 查询消息归档、审计日志、拦截记录，支持详情查看；导出能力仅保留为长期规划
4. **上线观测验证** — 上线初期提供临时观测窗口，查看最近少量完整 Chat 对话及 role 结构，用真实样本校验误拦截/误脱敏
5. **规则运营** — 查看规则、启停规则、观察命中效果，阶段 7 完整规则编辑暂不开发
6. **告警与审批** — 告警记录和审批处理暂不开发，仅保留入口或长期规划说明
7. **APIKey 映射与安全策略预留** — 为 APIKey 映射、上游 Key 替换、安全策略、多上游路由预留页面和接口字段；模型/token/资源权限由中转站管理
8. **系统配置** — 展示运行状态、Webhook、数据保留、健康检查、版本信息等配置

## 二、技术方案

### 2.1 技术选型

| 项目 | 方案 | 说明 |
|------|------|------|
| 前端形态 | 静态 HTML + 原生 JavaScript + CSS | MVP 阶段不引入 React/Vue，降低构建和部署复杂度 |
| 托管方式 | Nginx 静态文件 | `admin/static/` 挂载到 Nginx，API 由 FastAPI 提供 |
| API 前缀 | `/admin/api/*` | 所有管理接口统一前缀 |
| 认证方式 | Basic Auth + IP 白名单 | 阶段 4 使用轻量认证，阶段 10 可替换 SSO |
| 样式方案 | 单 CSS 文件 | `admin/static/css/style.css` |
| 公共逻辑 | 单 JS 文件 | `admin/static/js/app.js` |

### 2.2 目录结构

```text
admin/static/
├── index.html          # 仪表盘
├── login.html          # 表单登录页
├── blocks.html         # 拦截记录
├── archives.html       # 消息归档
├── observations.html   # 上线临时观测窗口
├── rules.html          # 规则管理
├── approvals.html      # 临时审批记录预留
├── api_keys.html       # APIKey 映射与上游 Key 替换
├── settings.html       # 系统设置
├── css/
│   └── style.css
└── js/
    └── app.js
```

当前阶段未单独创建 `audits.html` 与 `files.html`：安全审计能力由拦截记录页和后台 API 承载，文件安全页面随阶段 10 文件安全能力启用时补齐。

## 四、页面功能定义

### 4.1 仪表盘

**目标**：让管理员快速了解系统运行状态和安全态势。

| 模块 | 内容 |
|------|------|
| 核心指标卡 | 今日请求数、今日拦截数、今日告警数、活跃用户数、归档失败数 |
| 趋势图 | 最近 7 天请求趋势、拦截趋势、告警趋势 |
| Top 列表 | Top 命中规则、Top 风险用户、Top 模型 |
| 系统状态 | 服务状态、数据库状态、规则加载状态、Webhook 状态 |
| 阶段 6A 运行状态 | `/v1/*` 在途数、排队数、队列拒绝数、排队超时数、配置上限、容器总 in-flight/queue 指标、归档队列长度、上游连接池摘要、统计缓存时间；流式与非流式不拆分并发池 |
| 快捷入口 | 查看拦截记录、查看消息归档、查看规则、系统设置 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/stats` | 获取核心统计和趋势，阶段 6A 允许 5~15 秒短缓存 |
| `GET /admin/api/runtime` | 获取阶段 6A 单实例运行状态，当前已接入仪表盘 |
| `GET /admin/api/health` | 获取管理端展示用健康状态，当前已接入设置页 |
| `GET /admin/api/rules/top` | 获取 Top 命中规则，阶段 8 或后续增强接口，当前未实现且前端未调用 |

**验收标准**

- [ ] 页面加载后展示今日请求数、拦截数、告警数、活跃用户
- [ ] 趋势数据为空时展示空状态，不报错
- [ ] 后端异常时展示错误提示，不泄露堆栈信息

### 4.2 拦截记录

**目标**：查看所有 block/warn 安全事件，支持定位用户、规则和请求。

| 功能 | 说明 |
|------|------|
| 列表展示 | 时间、用户、APIKey 前后缀、模型、规则、级别、动作、请求 ID |
| 筛选 | 用户、规则、级别、模型、时间范围 |
| 详情 | 脱敏命中片段、请求哈希、关联归档、告警状态 |
| 跳转 | 跳转到消息归档详情、审计详情 |
| 导出 | 阶段 8 支持 CSV 导出 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/audits` | 查询 block/warn 事件 |
| `GET /admin/api/audits/{id}` | 查看事件详情 |
| `GET /admin/api/audits/export` | 导出审计记录，阶段 8 启用；当前未实现且前端未调用 |

**验收标准**

- [ ] 支持分页，默认每页 20 条，最大 100 条
- [ ] 命中片段默认脱敏展示
- [ ] 查询详情和导出行为写入管理员操作审计

### 4.3 消息归档

**目标**：查询完整对话记录，支持安全追溯和问题排查。

| 功能 | 说明 |
|------|------|
| 列表展示 | 时间、用户、APIKey 前后缀、模型、stream、是否拦截、耗时 |
| 筛选 | 用户、模型、是否拦截、关键词、时间范围 |
| 详情查看 | prompt、response、token、延迟、关联审计事件 |
| 脱敏策略 | 列表默认不展示 prompt/response 原文，详情页按权限展示 |
| 操作审计 | 查看完整 prompt/response 必须记录管理员操作日志 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/archives` | 分页查询消息归档 |
| `GET /admin/api/archives/{id}` | 查看归档详情 |
| `GET /admin/api/archives/stats` | 查询归档统计 |

**验收标准**

- [ ] 列表页不展示完整 prompt/response
- [ ] 详情页支持复制 request_id，不支持直接复制完整 APIKey
- [ ] 查看详情动作写入管理员操作审计

### 4.4 临时观测窗口

**目标**：上线初期用真实样本验证 Agent 消息组装、role 分布、误拦截和误脱敏情况，辅助决定是否调整规则或 role 策略。

| 功能 | 说明 |
|------|------|
| 最近记录 | 默认展示最近少量 Chat 归档记录，如最近 10 条，禁止默认全量加载 |
| 完整结构 | 展示原始 messages、role、content 类型、脱敏后 messages、响应、命中动作、命中规则、request_id |
| 对比查看 | 支持原始 messages 与脱敏后 messages 对照，突出被脱敏字段和参与 block 的最新允许 role message |
| 低干扰验证 | 标记是否发生 block/desensitize，并用于人工判断是否属于误拦截或误脱敏 |
| 安全边界 | 必须管理认证后访问；仅用于上线验证，后续由正式归档详情和审计详情替代 |
| 操作审计 | 查看完整原文、复制 request_id、导出样本等动作必须写入管理员操作审计 |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/observations/recent` | 查询最近少量完整 Chat 对话样本，阶段 3/4 上线初期启用 |
| `GET /admin/api/archives/{id}` | 复用归档详情接口查看单条完整记录 |
| `GET /admin/api/audits?request_id=...` | 查询关联命中事件 |

**验收标准**

- [ ] 默认只加载最近少量记录，不提供无条件全量浏览
- [ ] 每条记录能看到 messages 中的 role 和原始/脱敏前后差异
- [ ] 可识别当前策略实际参与 block 的 message 和参与 desensitize 的 role
- [ ] 页面顶部明确提示“上线临时观测用途，包含敏感原文，仅限管理员验证”
- [ ] 查看完整内容写入管理员操作审计

- [ ] 安全审计和管理员操作审计可切换查看

- [ ] 导出操作写入管理员操作审计

### 4.6 规则管理

**目标**：展示规则库状态和命中效果，支持后续规则运营。

| 功能 | v1.0 | v1.1+ |

**接口依赖**

| 接口 | 用途 |
|------|------|
| `GET /admin/api/rules` | 查询规则列表 |

| `POST /admin/api/rules/reload` | 手动触发规则热加载 |

**验收标准**

- [x] v1.0 支持查看规则列表和状态
- [x] v1.0 支持启用/停用单条规则，并立即触发热加载
- [x] v1.0 支持前端手动触发规则热加载
- [ ] 规则内容涉及敏感样例时默认脱敏展示
- [x] 启停、热加载动作写入管理员操作审计

### 4.8 APIKey 映射 / 安全策略 / 审批链

**目标**：阶段 5 启用 APIKey 映射管理，默认 K-Sync（前哨站 Key = 中转站 Key），支持手动录入历史 Key、加密存储、单条替换上游 Key、CSV 批量替换上游 Key；阶段 6 再接入 KeyProvider 调用中转站创建同名/同 owner Key 并获取 upstream_key；阶段 9 再绑定 SafetyHub 自身安全策略和审批链。模型权限、token 额度、速率限制和资源能力权限由中转站作为权威系统管理，SafetyHub 前端默认不配置、不判定。

| 功能 | 阶段 4 | 阶段 5 | 阶段 6 | 阶段 9 |
|------|--------|--------|--------|--------|
| APIKey 页面 | 只读占位 + 字段说明 | 创建、列表、吊销、手动录入、单条/批量替换 | 已支持手动录入/Provider 创建、reveal/复制完整 Key、Provider-aware 吊销 | 关联 SafetyHub 安全策略 + 关联审批链 |
| K-Sync | — | 默认启用，前哨站 Key = 中转站 Key | 保持兼容 | 保持兼容 |
| K-Decoupled | — | 上游 Key 被替换后自动进入 | Provider 创建/迁移时启用 | 保持兼容 |
| 模型/token/资源权限 | — | 中转站管理，SafetyHub 不配置 | 中转站 Provider 创建后在中转站侧配置和生效 | 同阶段 6 |
| 上游映射 | — | 手动维护上游 Key | Provider 自动创建/吊销/迁移 | 同阶段 6 |
| 中转站配额/速率观测（F20） | — | 不进入 SafetyHub schema | Provider 支持时只读展示或跳转中转站 | 阶段 10 可增强观测视图 |
| 安全策略（F18） | — | 字段预留 | 字段预留 | 创建策略 + 规则覆盖 + 继承 + 绑定到 Key |
| 审批链（F19） | — | 字段预留 | 字段预留 | 多级审批链 + 超时升级 + 不可审批高危规则 |

#### 4.7.2 阶段 5：K-Sync 默认 APIKey 管理

> 阶段 5 默认不要求中转站具备创建 APIKey 的接口。管理员可以把已有中转站 Key 手动录入 SafetyHub。默认模式下，客户端继续使用同一个 Key：它既是 SafetyHub Key，也是中转站 Key。

**列表页**

| 列 | 说明 |
|---|------|
| 名称 | 用户在创建时填写的备注名 |
| SafetyHub Key 前后缀 | K-Sync 下展示中转站 Key 前后缀；K-Decoupled 下展示 `sk-sh-` 前后缀 |
| 模式 | `K-Sync` / `K-Decoupled`，来自 `is_decoupled` |
| 中转站 Provider | 当前支持 `passthrough` / `static` / `oneapi_nanfu_yxai`；未来扩展 `openai_compat` |
| 中转站 Key 前后缀 | 永远不展示明文 |
| 资源权限状态 | 展示“由中转站管理”，可跳转中转站控制台或展示 Provider 引用信息 |
| 状态 | active / revoked / expired |
| 创建时间 / 过期时间 | — |
| 操作 | 显示/隐藏完整 Key、复制完整 Key、查看详情、吊销、替换上游 Key；显示/复制需调用受审计 reveal 接口 |

**创建表单（阶段 5 K-Sync 默认）**

| 字段 | 类型 | 说明 |
|------|------|------|
| 名称 | string | 必填，用户备注名 |
| 所属用户 | string | 必填，关联到 `user_id` |
| 中转站 Key | password | 必填；阶段 5 默认将它同时作为 SafetyHub Key 与 upstream_key |
| 资源权限提示 | readonly | 显示“模型权限、token 额度、速率限制和资源能力权限请在中转站管理” |
| 过期时间 | datetime | 可选 |
| Key 模式 | readonly | 默认 `K-Sync`，随机生成 SafetyHub Key 能力保留但默认隐藏 |

**创建流程（K-Sync）**

```text
管理员点击"新建 APIKey"
  → 输入已有中转站 Key
  → 前端 POST /admin/api/api-keys
  → 后端：
      1. key_hash = hash(中转站 Key)
      2. upstream_key_encrypted = encrypt(中转站 Key)
      3. key_prefix/key_suffix = 中转站 Key 前后缀
      4. is_decoupled = False
  → 客户端继续使用该中转站 Key 访问 SafetyHub
```

**详情抽屉**

| 区块 | 内容 |
|------|------|
| 基本信息 | 名称、所属用户、状态、Provider、K-Sync/K-Decoupled 模式 |
| Key 信息 | SafetyHub Key 前后缀 / 中转站 Key 前后缀；默认不展示明文，管理员可通过 reveal 接口按需显示/复制完整 SafetyHub Key并写入审计 |
| 资源权限 | 显示“由中转站管理”，SafetyHub 不展示或编辑模型/token/能力权限明细 |
| 上游替换 | "替换上游 Key"按钮 + 最近一次替换时间 + 替换人 |
| 关联记录 | 跳转到该 Key 的归档与审计列表 |
| 操作日志 | 创建、修改、吊销、替换等管理员操作审计 |

#### 4.7.3 阶段 5：上游 Key 替换 UI
