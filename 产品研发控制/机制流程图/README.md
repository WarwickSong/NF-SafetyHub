# 机制流程图

本目录用 Mermaid flowchart 从 6 个维度描述 SafetyHub 的运行机制，所有图均与当前代码实现对齐，可直接用于讲稿、培训、新人 onboarding 和架构评审。

| 维度 | 回答的问题 | 文件 |
|------|-----------|------|
| A. 请求透传链路 | 一条 `/v1/*` 请求从客户端进入到原样回到客户端经过了什么 | [A_请求透传链路.md](./A_请求透传链路.md) |
| B. 安全检测决策流 | 命中规则后系统怎么在 block / desensitize / pass 之间决策 | [B_安全检测决策流.md](./B_安全检测决策流.md) |
| C. 管理员管理逻辑 | 管理员从登录到操作 8 大模块的完整链路 | [C_管理员管理逻辑.md](./C_管理员管理逻辑.md) |
| D. APIKey 治理与上游 Key 替换流 | 用户 Key 与上游 Key 的替换、加密、轮转、吊销 | [D_APIKey治理与上游Key替换流.md](./D_APIKey治理与上游Key替换流.md) |
| E. 内网部署通信拓扑 | 部署到内网后办公网员工 → 前哨站 → 中转站 → LLM 厂商的整体通信 | [E_内网部署通信拓扑.md](./E_内网部署通信拓扑.md) |
| F. 数据异步归档与削峰流 | 归档/审计为什么不会拖慢主链路，数据怎么批量落库 | [F_数据异步归档与削峰流.md](./F_数据异步归档与削峰流.md) |

## 渲染方式

- VS Code / Trae：安装 `Markdown Preview Mermaid Support` 插件后直接预览。
- 飞书文档 / Notion / GitLab / GitHub：粘贴 ```mermaid 代码块即可渲染。
- 离线导出：用 `mmdc` (mermaid-cli) 渲染成 PNG / SVG。

## 阅读顺序建议

- **第一次接触系统**：A → B → E
- **运维/SRE 视角**：E → F → C
- **安全/合规视角**：B → D → C
- **二次开发视角**：A → F → D
