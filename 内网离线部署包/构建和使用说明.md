**目录职责说明**
- `内网离线部署包/`：应用层离线包产出区，负责生成 `safetyhub_intranet_bundle_*.tar.gz`，包内包含 SafetyHub 代码、Compose 配置、`.env`、业务镜像和依赖镜像。这个包默认假设目标服务器已经有 Docker / Docker Compose。
- `docker离线部署/`：整机离线部署工具区，负责准备 Docker Engine、docker compose v2、systemd 单元和最终 `docker-offline-deploy_*.tar.gz`。它会把 `内网离线部署包/` 生成的 `safetyhub_intranet_bundle_*.tar.gz` 作为输入，适用于目标服务器没有 Docker / Docker Compose 的场景。
- 推荐 Git 只管理脚本、配置、说明和 `.sha256`，不要提交 `*.tar`、`*.tar.gz`、Docker 二进制、镜像包和最终交付包。

**你可以在 `/var/www/NF-SafetyHub` 下生成一个完整“内网离线部署包”，上传这一个包到内网服务器，然后运行包里的 `install.sh`。**

**现在的最简流程**
- 在当前服务器生成离线包：
  
  ```bash
  cd /var/www/NF-SafetyHub
  bash 交付运行手册/build_intranet_offline_bundle.sh
  ```

- 生成位置默认是：
  
  ```text
  /var/www/NF-SafetyHub/内网离线部署包/safetyhub_intranet_bundle_YYYYMMDD_HHMMSS.tar.gz
  /var/www/NF-SafetyHub/内网离线部署包/safetyhub_intranet_bundle_YYYYMMDD_HHMMSS.tar.gz.sha256
  ```

- 把这两个文件复制到内网服务器 `/opt` 后执行：
  
  ```bash
  cd /opt
  sha256sum -c safetyhub_intranet_bundle_YYYYMMDD_HHMMSS.tar.gz.sha256
  tar -xzf safetyhub_intranet_bundle_YYYYMMDD_HHMMSS.tar.gz
  cd safetyhub_intranet_bundle_YYYYMMDD_HHMMSS
  bash install.sh
  ```

- 第一次运行会生成 `NF-SafetyHub/.env` 并提示你补齐密码/认证版本；补齐后再执行：
  
  ```bash
  bash install.sh
  ```

**离线包里包含**
- `safetyhub-images.tar`：包含三个镜像：
  - `nf-safetyhub-safetyhub:latest`
  - `postgres:16-alpine`
  - `nginx:1.27-alpine`
- `NF-SafetyHub/`：完整运行代码、Compose、Nginx 配置和交付脚本。
- `data/safetyhub_apikeys_YYYYMMDD_HHMMSS.sql`：只包含 APIKeys。
- `install.sh`：内网服务器一键安装入口。

**脚本会自动做什么**
- 自动执行 `docker load -i safetyhub-images.tar`。
- 自动创建 `.env`，并把 `SAFETYHUB_SKIP_BUILD=true` 写进去。
- 检查 `.env` 是否还有占位符。
- 自动启动 PostgreSQL、SafetyHub、Nginx。
- 自动清空目标库运行数据表。
- 自动导入 APIKeys。
- 最后输出 `docker compose ps` 和健康检查结果。
